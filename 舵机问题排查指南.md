# 舵机问题排查指南

## 🔴 症状描述
- LCD验证成功
- 舵机基本不动或完全不动

---

## 📋 排查步骤

### ✅ 步骤1：检查舵机电源（最常见问题）

#### 问题描述
舵机需要**5V电源**和**足够大的电流**（通常200-500mA）。STM32开发板的3.3V输出无法驱动舵机！

#### 检查方法
```
舵机引脚定义（通常）：
┌─────────────────┐
│  橙/黄  - 信号线  │ → PA0
│  红     - 电源线  │ → 外部5V电源
│  棕/黑  - 地线    │ → GND（共地）
└─────────────────┘
```

#### 解决方案
1. **使用独立5V电源**（推荐）
   ```
   红线 → 5V电源正极
   棕/黑线 → 5V电源负极（同时连接STM32的GND）
   橙/黄线 → PA0
   ```

2. **使用USB 5V**（临时测试）
   ```
   红线 → USB 5V（红线）
   棕/黑线 → USB GND（黑线，同时连接STM32的GND）
   橙/黄线 → PA0
   ```

#### ⚠️ 重要提示
**必须共地！** 舵机的地线必须与STM32的GND连接！

---

### ✅ 步骤2：验证PWM信号输出

#### 方法1：使用示波器（推荐）
- 测量PA0引脚
- **预期波形**：
  - 周期：20ms（50Hz）
  - 高电平宽度：1.5ms（初始值）
  - 电平：0-3.3V

#### 方法2：使用LED指示
如果PA0连接LED，应该能看到LED闪烁（但人眼可能看不到20ms周期的闪烁）

#### 方法3：使用万用表测量平均电压
- PA0平均电压 ≈ 1.5ms / 20ms × 3.3V ≈ 0.25V
- 如果电压为0V或3.3V，说明PWM未输出

---

### ✅ 步骤3：检查硬件连接

#### 检查清单
- [ ] 舵机信号线（橙/黄）已连接到PA0
- [ ] 舵机电源线（红）已连接到5V电源
- [ ] 舵机地线（棕/黑）已连接到GND
- [ ] **STM32的GND与舵机的GND已共地**
- [ ] 舵机型号正确（标准舵机，不是连续旋转舵机）

#### 常见错误
❌ 舵机红线连接到STM32的3.3V → **电源不足**
❌ 舵机地线未与STM32共地 → **信号回路不通**
❌ 舵机连接到错误的引脚 → **无PWM输出**

while(1)

#### 预期现象
舵机应该每秒转动一次，来回运动：
```
0° → 90° → 180° → 90° → 0° → ...
```

### ✅ 步骤5：验证PWM配置参数

#### STM32F407 PWM参数计算
```
系统时钟：168MHz
APB1时钟：42MHz
TIM2时钟：84MHz（APB1 × 2，因为APB1预分频器≠1）

预分频：84-1 = 83
计数频率：84MHz / 84 = 1MHz（1us精度）

PWM周期：20000个计数 = 20ms = 50Hz ✓
脉宽范围：500-2500us
初始脉宽：1500us（90度）
```

#### 如何验证配置
在Keil中调试模式：
1. 打开Peripherals → Timers → TIM2
2. 检查以下寄存器：
   - `PSC`（预分频）= 83
   - `ARR`（自动重装值）= 19999
   - `CCR1`（捕获/比较值）= 1500
   - `CCMR1`的OC1M位应为PWM模式

## 🔧 高级调试方法

### 方法1：使用逻辑分析仪
- 连接PA0到逻辑分析仪
- 观察PWM波形是否正确输出

### 方法2：修改PWM频率测试
如果舵机不响应，尝试降低PWM频率：

修改`pwm.h`：
```c
#define PWM_PERIOD 40000  // 改为40ms（25Hz）测试
```

### 方法3：直接测试GPIO输出
修改代码，直接控制PA0为高电平，验证舵机是否工作：

在`main.c`的初始化后添加：
```c
// 测试GPIO输出
GPIO_SetBits(GPIOA, GPIO_Pin_0);  // PA0输出高电平
delay_ms(1000);
GPIO_ResetBits(GPIOA, GPIO_Pin_0);  // PA0输出低电平
delay_ms(1000);
```

## 📊 问题分类与解决方案

### 问题1：舵机完全不动
**可能原因**：
1. ❌ 电源未连接或电源不足
2. ❌ 地线未共地
3. ❌ 舵机损坏

**解决方案**：
- 检查电源连接
- 确保共地
- 更换舵机测试

### 问题2：舵机轻微抖动但不转动
**可能原因**：
1. ❌ 电源电流不足
2. ❌ PWM频率不正确
3. ❌ 舵机扭矩不足

**解决方案**：
- 使用更大电流的电源（至少500mA）
- 检查PWM频率（应为50Hz）
- 检查舵机负载

### 问题3：舵机只转动一次
**可能原因**：
1. ❌ 程序卡死
2. ❌ PWM配置错误

**解决方案**：
- 检查main.c的while(1)循环
- 使用调试模式单步执行

### 问题4：舵机转动角度不正确
**可能原因**：
1. ❌ PWM脉宽范围不匹配
2. ❌ 舵机型号差异

**解决方案**：
- 调整`PWM_MIN`和`PWM_MAX`的值
- 尝试不同的脉宽范围（如600-2400us）

## 🎯 推荐测试流程

### 测试1：基础电源测试
```
1. 舵机红线 → 5V
2. 舵机棕/黑线 → GND
3. 不连接信号线
4. 上电后舵机应能自由转动（手动转动）
```

### 测试2：GPIO控制测试
```
1. PA0直接输出高电平
2. 舵机应转到0度位置
3. PA0输出低电平
4. 舵机应保持位置
```

### 测试3：PWM输出测试
```
1. 运行修改后的程序
2. 舵机应来回转动（0° → 90° → 180° → 90°）
```

## ✅ 最可能的原因（按概率排序）

1. **90% - 电源问题**：舵机未连接独立5V电源
2. **80% - 共地问题**：STM32和舵机地线未连接
3. **5% - 舵机损坏**：舵机本身故障
4. **5% - 引脚连接错误**：信号线连接到错误的引脚

**请按照上述步骤逐一排查，特别注意电源和共地问题！**
