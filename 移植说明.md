# 智能门锁代码移植说明 (CH32V30x → STM32F407VET6)

## 📋 移植概述
本项目从沁恒CH32V30x平台移植到STM32F407VET6平台，保留了核心功能代码，修改了底层驱动。

## 🔧 引脚映射对照表

### 1. PWM舵机控制
| 功能 | CH32V30x | STM32F407VET6 | 说明 |
|------|----------|---------------|------|
| 舵机PWM | TIM2_CH1 (PA0) | TIM2_CH1 (PA0) | ✅ 引脚保持不变 |

**配置差异：**
- CH32V30x: 96MHz主频，预分频96-1 → 1MHz (1us精度)
- STM32F407: 168MHz主频，APB1=42MHz，TIM2时钟=84MHz，预分频84-1 → 1MHz (1us精度)

### 2. LCD ST7735显示屏 (SPI3)
| 功能 | CH32V30x | STM32F407VET6 | 说明 |
|------|----------|---------------|------|
| SPI_SCK | PB3 (SPI3_SCK) | **PC10 (SPI3_SCK)** | ⚠️ 引脚变更 |
| SPI_MOSI | PB5 (SPI3_MOSI) | **PC12 (SPI3_MOSI)** | ⚠️ 引脚变更 |
| SPI_MISO | PB4 | **PC11 (SPI3_MISO)** | 可选 |
| LCD_CS | PD4 | **PA4** | ⚠️ 引脚变更（避免USART冲突） |
| LCD_DC | PD3 | **PA3** | ⚠️ 引脚变更（避免USART冲突） |
| LCD_BL | PD5 | **PB9** | ⚠️ 引脚变更（避免USART冲突） |
| LCD_RES | PB4 | **PA1** | ⚠️ 引脚变更（避免USART冲突） |

**关键修改：**
- STM32F407的SPI3引脚在PC10/PC11/PC12
- LCD控制引脚重新映射，避免与USART2冲突：
  - LCD_CS: PA4（原PD4）
  - LCD_DC: PA3（原PD3）
  - LCD_BL: PB9（原PD5）
  - LCD_RES: PA1（原PB4）
- 需要配置GPIO复用功能：`GPIO_PinAFConfig(GPIOC, GPIO_PinSource10, GPIO_AF_SPI3)`

## 🚀 关键移植修改

### 1. GPIO初始化差异
**CH32V30x:**
```c
RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA, ENABLE);
GPIO_InitStructure.GPIO_Mode = GPIO_Mode_AF_PP;
GPIO_InitStructure.GPIO_Speed = GPIO_Speed_50MHz;
```

**STM32F407:**
```c
RCC_AHB1PeriphClockCmd(RCC_AHB1Periph_GPIOA, ENABLE);
GPIO_InitStructure.GPIO_Mode = GPIO_Mode_AF;
GPIO_InitStructure.GPIO_Speed = GPIO_Speed_100MHz;
GPIO_InitStructure.GPIO_OType = GPIO_OType_PP;
GPIO_InitStructure.GPIO_PuPd = GPIO_PuPd_UP;
GPIO_PinAFConfig(GPIOA, GPIO_PinSource0, GPIO_AF_TIM2);
```

### 2. 定时器时钟差异
- **CH32V30x**: 系统时钟96MHz，TIM2时钟96MHz
- **STM32F407**: 系统时钟168MHz，APB1时钟42MHz，TIM2时钟84MHz

**预分频计算：**
- CH32V30x: `TIM_Prescaler = 96 - 1` (96MHz/96 = 1MHz)
- STM32F407: `TIM_Prescaler = 84 - 1` (84MHz/84 = 1MHz)

### 3. SPI引脚复用
STM32F407需要显式配置GPIO复用功能：
```c
GPIO_PinAFConfig(GPIOC, GPIO_PinSource10, GPIO_AF_SPI3);  // SCK
GPIO_PinAFConfig(GPIOC, GPIO_PinSource11, GPIO_AF_SPI3);  // MISO
GPIO_PinAFConfig(GPIOC, GPIO_PinSource12, GPIO_AF_SPI3);  // MOSI
```

### 4. 延时函数差异
- **CH32V30x**: `Delay_Init()` 和 `Delay_Ms()`
- **STM32F407**: `delay_init(168)` 和 `delay_ms()`

## 📝 注意事项

1. **时钟配置**：
   - STM32F407主频168MHz，delay_init参数必须为168
   - APB1时钟42MHz，定时器时钟为APB1的2倍（84MHz）

2. **GPIO速度**：
   - STM32F407支持100MHz GPIO速度，建议使用高速配置

3. **SPI速度**：
   - SPI3时钟分频：`SPI_BaudRatePrescaler_4`
   - APB1=42MHz，SPI3时钟=42MHz/4=10.5MHz

4. **字库说明**：
   - 包含ASCII字库（8x16和16x32）
   - 包含中文汉字字库（16x16, 24x24, 32x32）
   - 自带部分汉字：正、在、启、动、成、功等

5. **USART资源保护**：
   - LCD控制引脚已重新映射，避免与USART冲突
   - 保留所有6个USART资源：
     - USART1: PA9/PA10
     - USART2: PA2/PA3
     - USART3: PB10/PB11
     - UART4: PA0/PA1
     - UART5: PC12/PD2
     - USART6: PC6/PC7
