## User Requirements

- 将“第二个项目（矩阵键盘密码门锁）”完整功能迁移到 STM32F407ZGT6 工程模板中。
- 键盘引脚固定为：行 PD11/PD9/PE15/PE13，列 PE11/PE9/PE7/PC5。
- 密码与按键规则保持一致：密码 277516，A 键清空输入，D 键执行确认校验。
- 交付范围包含：代码移植完成，以及一份可直接上板执行的验证清单。

## Product Overview

- 上电后保留原有开机动画与主页显示流程，进入“门已上锁，请输入密码”界面。
- 用户通过 4x4 矩阵键盘输入 6 位密码，屏幕输入区以“*”动态显示已输入位数。
- 确认后进行密码比对：正确则开锁，错误则保持上锁；清空键可随时重置输入。

## Core Features

- 矩阵键盘扫描与按键事件处理（数字录入、清空、确认）。
- 密码缓存与长度控制（最多 6 位）及比对判定。
- 门锁状态控制（上锁/开锁）与舵机动作联动。
- 周期任务调度（按键高频扫描、界面低频刷新）与稳定主循环运行。
- 上板验证流程：覆盖上电、输入、清空、错误密码、正确密码等关键场景。

## Tech Stack Selection

- 复用现有 STM32F4 标准库工程（StdPeriph + Keil 工程结构），不引入新框架。
- 继续沿用现有模块：
- `HARDWARE/LCD`（显示）
- `HARDWARE/PWM`（舵机）
- `USER/main.c`（业务主流程）
- `USER/stm32f4xx_it.c`（中断）
- 新增独立键盘驱动模块，保持与现有 `HARDWARE` 分层一致。

## Implementation Approach

- 采用“驱动层迁移 + 业务层改写 + 中断节拍支撑”策略：先落地矩阵键盘 GPIO 扫描，再将第二项目的密码状态机与任务调度迁入 `main.c`，最后接入 TIM3 1ms 节拍中断。
- 关键决策：
- 保留原模板的 LCD 启动视觉流程，仅替换主循环业务，降低改动风险。
- 复用 `pwm.c` 的 `Servo_SetAngle`，新增统一 `lock(enable)` 封装，避免业务层直接写比较寄存器。
- 将按键扫描周期设为 10ms、LCD 刷新 100ms，避免阻塞式延时导致漏键或闪烁。
- 性能与可靠性：
- 按键扫描与界面刷新均为 O(1) 固定开销；主循环仅遍历少量任务表，复杂度 O(n)，n 很小（2~3）。
- 通过边沿检测（key_down）防止长按重复录入，输入上限保护防止数组越界。
- 失败默认保持上锁，保证异常情况下安全状态可控。

## Implementation Notes

- 严格使用已确认键盘引脚：行输出 `PD11/PD9/PE15/PE13`，列输入上拉 `PE11/PE9/PE7/PC5`。
- `TIM3_IRQHandler` 放在 `USER/stm32f4xx_it.c`，只做节拍计数与中断清标志，避免中断内做重逻辑。
- 迁移 `string_check` 时修正比较细节并加长度边界，防止隐性逻辑错误。
- 保持向后兼容：不改 LCD/PWM 已有接口语义，仅增量扩展；避免无关重构。
- Keil 工程需同步加入新增源文件与头文件搜索路径，确保可直接编译。

## Architecture Design

- 结构关系：
- `KEY` 驱动负责矩阵扫描与键值输出。
- `main` 负责密码状态机、任务调度、UI刷新、锁控调用。
- `TIM3` 中断提供 1ms 系统节拍。
- `PWM` 提供开锁/上锁执行动作。
- `LCD` 提供输入反馈与状态文案显示。
